## 1. 项目概述

### 1.1 任务重述

基于"众爱联盟公益积分微信小程序"产品需求文档（PRD），设计一套完整的前后端技术方案：

| 维度 | 说明 |
|------|------|
| **目标** | 为公益组织打造"双积分制"（活动积分 + 捐助积分）志愿者管理平台 |
| **前端** | Uniapp + wot-starter-v2 模板，编译目标为微信小程序 + H5 |
| **后端** | Nuxt v4 提供 API 服务 + 后台管理界面，UI 使用 Nuxt UI v4 |
| **数据库** | PostgreSQL |
| **输出** | Markdown 格式的详细设计文档 |

## 3. 数据库设计

### 3.1 ER 关系概览

```
users ──┬── activity_registrations ── activities
        ├── activity_checkins ─────── activities
        ├── donations
        ├── point_transactions
        ├── market_posts
        └── mall_orders ── mall_order_items ── mall_products

activities ── activity_registrations
           ── activity_checkins
           ── activity_galleries

point_rules (独立配置表)
banners (独立配置表)
```

### 3.2 数据表详细设计

#### 3.2.1 用户表 `users`

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | UUID | PK | 用户ID |
| `openid` | VARCHAR(64) | UNIQUE, NOT NULL | 微信OpenID |
| `union_id` | VARCHAR(64) | UNIQUE | 微信UnionID |
| `phone` | VARCHAR(20) | UNIQUE | 手机号 |
| `nickname` | VARCHAR(50) | | 微信昵称 |
| `avatar_url` | VARCHAR(500) | | 头像URL |
| `real_name` | VARCHAR(50) | | 真实姓名 |
| `id_card_no` | VARCHAR(18) | | 身份证号（加密存储） |
| `real_name_verified` | BOOLEAN | DEFAULT false | 实名认证状态 |
| `member_no` | VARCHAR(20) | UNIQUE | 会员编号（如 ZA-00001） |
| `role` | VARCHAR(20) | DEFAULT 'volunteer' | 角色：volunteer / leader / admin |
| `status` | VARCHAR(20) | DEFAULT 'active' | 状态：active / frozen |
| `honor_level` | INT | DEFAULT 0 | 荣誉等级 |
| `created_at` | TIMESTAMPTZ | DEFAULT now() | 注册时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT now() | 更新时间 |

**索引**: `idx_users_openid`, `idx_users_role`, `idx_users_member_no`

#### 3.2.2 积分账户表 `point_accounts`

| 字段名 | 类型 | 约束 | 说明 |

### 4.3 核心业务逻辑设计

#### 4.3.1 积分发放流程

```
活动积分发放流程：
1. 志愿者GPS签到 / 领队扫码确认
2. 记录 activity_checkins 表
3. 领队点击"完成活动"，选择确认的签到记录
4. 系统批量处理：
   a. 更新 activity_checkins.points_granted = true
   b. 更新 point_accounts.activity_points_balance += N
   c. 更新 point_accounts.activity_points_total += N
   d. 插入 point_transactions（change_type=earn）

捐助积分发放流程：
1. 志愿者提交捐助凭证
2. 管理员审核凭证
3. 审核通过时：
   a. 根据积分规则计算积分数
   b. 更新 donations.status = 'approved', points_granted = true
   c. 更新 point_accounts.donation_points_balance += N
   d. 更新 point_accounts.donation_points_total += N
   e. 插入 point_transactions（change_type=earn）
```

#### 4.3.2 积分消耗流程

```
爱心集市发布帖子：
1. 用户选择帖子类型和使用的积分类型
2. 查询积分规则获取所需积分数
3. 检查用户对应积分余额是否充足
4. 开启数据库事务：
   a. 扣减 point_accounts 对应余额
   b. 插入 point_transactions（change_type=spend, amount=-N）
   c. 创建 market_posts 记录
5. 提交事务

积分商城兑换：
1. 用户选择商品和使用的积分类型
2. 计算总积分消耗
3. 检查余额
4. 开启数据库事务：
   a. 扣减 point_accounts 对应余额
   b. 插入 point_transactions（change_type=spend）
   c. 创建 mall_orders + mall_order_items
   d. 更新 mall_products.stock_remaining
5. 提交事务
```

#### 4.3.3 签到验证逻辑

```typescript
// GPS签到验证伪代码
async function verifyGpsCheckin(activityId: string, userId: string, lat: number, lng: number) {
  const activity = await getActivity(activityId)

  // 1. 检查活动状态
  if (activity.status !== 'ongoing') throw new Error('活动未在进行中')

  // 2. 检查时间范围
  const now = new Date()
  if (now < activity.startTime || now > activity.endTime) {
    throw new Error('不在活动时间范围内')
  }

  // 3. 检查是否已报名
  const registration = await getRegistration(activityId, userId)
  if (!registration || registration.status !== 'approved') {
    throw new Error('未报名或报名未通过')
  }

  // 4. 检查是否已签到
  const existing = await getCheckin(activityId, userId)
  if (existing) throw new Error('已签到，请勿重复签到')

  // 5. GPS距离验证（使用Haversine公式）
  const distance = calculateDistance(lat, lng, activity.latitude, activity.longitude)
  if (distance > activity.checkinRadius) {
    throw new Error(`距离活动地点${distance.toFixed(0)}米，超出签到范围`)
  }

  // 6. 创建签到记录
  return await createCheckin({
    activityId, userId,
    checkinType: 'gps',
    latitude: lat,
    longitude: lng,
  })
}
```

### 5.4 前端请求封装

```typescript
// utils/request.ts
const BASE_URL = import.meta.env.VITE_API_BASE_URL

interface RequestOptions {
  url: string
  method?: 'GET' | 'POST' | 'PUT' | 'PATCH' | 'DELETE'
  data?: any
  header?: Record<string, string>
  showLoading?: boolean
  showError?: boolean
}

export function request<T = any>(options: RequestOptions): Promise<T> {
  return new Promise((resolve, reject) => {
    const { url, method = 'GET', data, header = {}, showLoading = true, showError = true } = options

    if (showLoading) {
      uni.showLoading({ title: '加载中...' })
    }

    // 自动附加 Token
    const token = uni.getStorageSync('token')
    if (token) {
      header['Authorization'] = `Bearer ${token}`
    }

    uni.request({
      url: `${BASE_URL}${url}`,
      method,
      data,
      header: { 'Content-Type': 'application/json', ...header },
      success: (res) => {
        const result = res.data as ApiResponse<T>
        if (result.code === 0) {
          resolve(result.data)
        } else if (result.code === 401) {
          // Token 过期，跳转登录
          const userStore = useUserStore()
          userStore.logout()
          uni.navigateTo({ url: '/pages/login/index' })
          reject(new Error('登录已过期'))
        } else {
          if (showError) {
            uni.showToast({ title: result.message, icon: 'none' })
          }
          reject(new Error(result.message))
        }
      },
      fail: (err) => {
        if (showError) {
          uni.showToast({ title: '网络异常', icon: 'none' })
        }
        reject(err)
      },
      complete: () => {
        if (showLoading) uni.hideLoading()
      },
    })
  })
}
```

## 6. 接口规范与安全设计

### 6.1 接口规范

#### 6.1.1 请求头规范

| Header | 说明 | 示例 |
|--------|------|------|
| `Content-Type` | 请求体格式 | `application/json` |
| `Authorization` | JWT Token | `Bearer eyJhbGciOi...` |
| `X-Platform` | 客户端平台 | `wechat-mp` / `h5` |
| `X-Version` | 客户端版本 | `1.0.0` |

#### 6.1.2 错误码定义

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或Token过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如重复报名） |
| 422 | 业务校验失败 |
| 500 | 服务器内部错误 |

#### 6.1.3 分页规范

请求参数：`page`（页码，从1开始）、`pageSize`（每页条数，默认10，最大50）

### 6.2 安全设计

| 安全措施 | 实现方式 |
|----------|----------|
| **认证** | JWT + Refresh Token 双Token机制 |
| **微信登录** | wx.login → 后端code2session → 生成JWT |
| **权限控制** | 服务端中间件校验角色权限 |
| **数据加密** | 身份证号 AES 加密存储 |
| **接口限流** | Nitro 内置 rate limiting |
| **SQL注入防护** | Drizzle ORM 参数化查询 |
| **XSS防护** | 输入过滤 + 输出转义 |
| **文件上传** | 限制文件类型、大小，图片压缩 |
| **HTTPS** | 全站 HTTPS |
| **CORS** | Nuxt routeRules 配置允许的来源 |

### 6.3 文件上传设计

```
上传流程：
1. 前端选择图片/视频 → uni.chooseImage / uni.chooseVideo
2. 前端压缩图片 → uni.compressImage（视频由后端转码压缩）
3. 前端上传 → POST /api/upload
4. 后端接收 → 校验类型/大小 → 存储到本地/OSS
5. 返回文件URL → 前端使用

支持的文件类型：
- 图片：jpg, jpeg, png, gif, webp（最大5MB）
- 视频：mp4, mov（最大100MB，活动视频/照片使用）
- 文档：pdf（最大10MB，用于凭证）
```
